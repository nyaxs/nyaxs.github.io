首先要检查版本对应关系

MyBatis-Spring-Boot-Starter 要求如下依赖版本:

MyBatis-Spring-Boot-Starter MyBatis-Spring Spring Boot Java
2.1 2.0 (need 2.0.2+ for enable all features) 2.1 or higher 8 or higher
1.3 1.3 1.5 6 or higher

Maven
如果您使用的是Maven，只需将以下依赖项添加到pom.xml中：

<依赖性>
org.mybatis.spring.boot </ groupId>
mybatis-spring-boot-starter </ artifactId>
2.1.3 </ version>
</ dependency>
摇篮
如果使用gradle将其添加到您的build.gradle：

依赖项{
编译（“ org.mybatis.spring.boot：mybatis-spring-boot-starter：2.1.3”）
}
快速设置
您可能已经知道，要将MyBatis与Spring一起使用，您至少需要一个SqlSessionFactory和至少一个mapper接口。

MyBatis-Spring-Boot-Starter将：

自动检测现有的数据源
将创建并注册一个实例的SqlSessionFactory是通过数据源作为输入使用SqlSessionFactoryBean会
将创建并注册从SqlSessionFactory中获取的SqlSessionTemplate的实例
自动扫描您的映射器，将它们链接到SqlSessionTemplate并将它们注册到Spring上下文，以便可以将它们注入到您的bean中
假设我们有以下映射器：

@Mapper
公共接口CityMapper {

@Select（“ SELECT * FROM CITY WHERE state =＃{state}”）
城市findByState（@Param（“ state”）字符串状态）；

}
您只需要创建一个普通的Spring引导应用程序，然后按如下所示注入映射器即可（在Spring 4.3+上可用）：

@SpringBootApplication
公共类SampleMybatisApplication实现CommandLineRunner {

私人最终CityMapper cityMapper；

公共SampleMybatisApplication（CityMapper cityMapper）{
this.cityMapper = cityMapper;
}

公共静态void main（String [] args）{
SpringApplication.run（SampleMybatisApplication.class，args）;
}

@Override
public void run（String … args）引发异常{
System.out.println（this.cityMapper.findByState（“ CA”））;
}

}
这就是您要做的。您的应用程序现在可以作为普通的Spring Boot应用程序运行。

进阶扫描
默认情况下，MyBatis-Spring-Boot-Starter将搜索带有@Mapper批注的映射器。

您可能需要指定自定义注释或标记界面进行扫描。如果是这样，则必须使用@MapperScan批注。在MyBatis-Spring参考页面中查看有关它的更多信息。

如果MyBatis-Spring-Boot-Starter 在Spring的上下文中找到至少一个MapperFactoryBean，则它将不会启动扫描过程，因此，如果您想完全停止扫描，则应使用@Bean方法显式注册映射器。

使用SqlSession
创建了SqlSessionTemplate的实例并将其添加到Spring上下文中，因此您可以使用MyBatis API使其像下面那样注入到您的bean中（在Spring 4.3+上可用）：

@零件
公共类CityDao {

私有最终SqlSession sqlSession;

公共CityDao（SqlSession sqlSession）{
this.sqlSession = sqlSession;
}

公共城市selectCityById（长ID）{
返回this.sqlSession.selectOne（“ selectCityById”，id）;
}

}
组态
与其他任何Spring Boot应用程序一样，MyBatis-Spring-Boot-Application配置参数存储在application.properties（或application.yml）中。

MyBatis使用前缀mybatis作为其属性。

可用属性为：

属性 描述
配置位置 MyBatis xml配置文件的位置。
检查配置位置 指示是否执行MyBatis xml配置文件的存在性检查。
映射器位置 Mapper xml配置文件的位置。
类型别名包 用于搜索类型别名的软件包。（包定界符为“ ，; \ t \ n ”）
类型别名超类型 用于过滤类型别名的超类。如果未指定，则MyBatis将所有从type-aliases-package搜索的类当作类型别名。
类型处理程序包 用于搜索类型处理程序的软件包。（包定界符为“ ，; \ t \ n ”）
执行者类型 执行程序类型：SIMPLE，REUSE，BATCH
默认脚本语言驱动程序 默认脚本语言驱动程序类。此功能需要与mybatis-spring 2.0.2+一起使用。
配置属性 MyBatis配置的外部化属性。可以将指定的属性用作MyBatis配置文件和Mapper文件上的占位符。有关详细信息，请参见MyBatis参考页。
延迟初始化 是否启用映射器bean的延迟初始化。设置为true可启用延迟初始化。此功能需要与mybatis-spring 2.0.2+一起使用。
组态。* MyBatis Core提供的Configuration bean的属性键。关于可用的嵌套属性，请参见MyBatis参考页。注意：此属性不能与config-location一起使用。
scripting-language-driver.thymeleaf。* MyBatis Thymeleaf提供的ThymeleafLanguageDriverConfig bean的属性键。有关可用的嵌套属性，请参见MyBatis Thymeleaf参考页。
scripting-language-driver.freemarker。* MyBatis FreeMarker提供的FreeMarkerLanguageDriverConfig bean的属性键。关于可用的嵌套属性，请参见MyBatis FreeMarker参考页。此功能需要与mybatis-freemarker 1.2.0+一起使用。
scripting-language-driver.velocity。* MyBatis Velocity提供的VelocityLanguageDriverConfig bean的属性键。关于可用的嵌套属性，请参见MyBatis Velocity参考页。此功能需要与mybatis-velocity 2.1.0+一起使用。

使用ConfigurationCustomizer
MyBatis-Spring-Boot-Starter提供了自定义使用Java Config自动配置生成的MyBatis配置的机会。MyBatis-Spring-Boot-Starter将自动搜索实现ConfigurationCustomizer接口的bean ，并调用自定义MyBatis配置的方法。（自1.2.1或更高版本开始可用）

例如：

@组态
公共类MyBatisConfig {
@豆
ConfigurationCustomizer mybatisConfigurationCustomizer（）{
返回新的ConfigurationCustomizer（）{
@Override
public void custom（配置配置）{
//定制…
}
};
}
}
使用SpringBootVFS
MyBatis-Spring-Boot-Starter提供SpringBootVFS作为VFS的实现类。该VFS是用于从应用程序（或应用服务器）搜索类别（例如目标类型的别名，类型处理程序类的）。如果使用可执行jar运行Spring Boot应用程序，则需要使用SpringBootVFS。MyBatis-Spring-Boot-Starter提供的自动配置功能会自动使用它，但手动配置不会自动使用它（例如，使用多个DataSource时）。

如何在手动配置上使用SpringBootVFS：

@组态
公共类MyBatisConfig {
@豆
公共SqlSessionFactory masterSqlSessionFactory（）引发异常{
SqlSessionFactoryBean factoryBean =新的SqlSessionFactoryBean（）;
factoryBean.setDataSource（masterDataSource（））;
factoryBean.setVfs（SpringBootVFS.class）; //将SpringBootVFS类设置为SqlSessionFactoryBean
// …
返回factoryBean.getObject（）;
}
}
